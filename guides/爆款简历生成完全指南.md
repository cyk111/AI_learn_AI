# 🎯 爆款简历生成完全指南

> **核心原则：宁缺毋滥，高端大气有品位。让面试官 6 秒内说 Yes。**
> 拒绝模板感、拒绝空话、拒绝信息堆砌。
> 追求：精准用词 × 量化成果 × 设计呼吸感 × ATS 友好

---

## 一、模型选择

| 模型 | 简历生成能力 | 最佳用途 | 推荐度 |
|------|:-----------:|----------|:------:|
| **Claude Opus 4.8** | ⭐⭐⭐⭐⭐ | 深度改写、叙事打磨、跨格式生成 | 🥇 首选 |
| **Claude Sonnet 4.5** | ⭐⭐⭐⭐⭐ | 快速生成、ATS 匹配、DOCX 输出 | 🥇 日常首选 |

### 🥇 为什么 Claude 是首选？

1. **原生 DOCX/PDF 生成**（2025年9月上线）：对话中直接生成 .docx 简历文件
2. **超长上下文**（200K tokens）：一次吃进你的完整履历 + 10 个 JD
3. **写作质量最高**：改写后的简历 bullet point 比 GPT 更自然、更精准
4. **跨格式互转**：上传旧简历 PDF → 分析 → 生成新 DOCX

## 二、技术栈：排版引擎 PK

### RenderCV（🥇 首选，12,000+ GitHub Stars）

```bash
pip install "rendercv[full]"
rendercv new "Your Name"          # 生成 YAML 模板
# 编辑 YAML → rendercv render "Your_Name_CV.yaml"  # 生成 PDF
```

| 优势 | 说明 |
|------|------|
| **YAML 驱动** | 内容和排版分离，LLM 极易生成/修改 |
| **Typst 引擎** | 替代 LaTeX，排版质量碾压 Word/Google Docs |
| **JSON Schema 校验** | 自动检测缺失字段、格式错误 |
| **多主题** | classic / moderncv / engineeringresumes / sb2nov |
| **多格式输出** | PDF / PNG / HTML / Markdown / Typst |
| **Git 友好** | YAML 纯文本可 diff、merge、tag |

### claude-cv-factory（RenderCV 增强版）

```bash
git clone https://github.com/yurictl/claude-cv-factory
cd claude-cv-factory
pip install -r requirements.txt
python3 render_cv.py
```

额外特性：YAML 银行（cv-bank/）、自动校验（3 种模式）、一键多格式渲染

### 其他方案

| 方案 | 适用场景 | 局限 |
|------|----------|------|
| resumake-mcp | Claude Desktop 直接生成 LaTeX PDF | 需 MCP 配置 |
| JSON Resume | 标准 Schema，生态丰富 | 排版不如 RenderCV |
| Claude 原生 DOCX | 最快，直接对话生成 | 排版精细度不如 Typst |
| LaTeX (moderncv) | 学术/欧洲求职 | 学习成本高 |

## 三、操作流程：4 阶段法（总耗时 25-35 分钟）

```
Phase 1: 数据提取 (5 min)    → 旧简历/脑暴 → 结构化 YAML
Phase 2: 内容打磨 (10 min)   → STAR 改写 → 量化成果 → 删除废话
Phase 3: ATS 优化 (5 min)    → JD 匹配 → 关键词缺口分析
Phase 4: 渲染输出 (5 min)    → RenderCV 生成 PDF → 人工审校
```

## 四、Prompt 模板

### 🅰️ Phase 1: 数据提取 → 结构化 YAML

```text
你是一位顶级职业顾问，曾帮助 500+ 人进入 FAANG、MBB 和独角兽公司。

请根据我提供的工作经历，生成一份 RenderCV 兼容的 YAML 简历文件。

【我的背景】
{粘贴旧简历、LinkedIn 信息、或自由叙述你的经历}

【目标职位】{职位名称}
【目标行业】{行业}
【简历语言】{中文 / 英文 / 中英双语}

【YAML 结构要求】
请严格按照以下结构输出，每个字段都不能遗漏：

```yaml
cv:
  name: {全名}
  label: {当前职位或目标职位，如 "Senior Frontend Engineer"}
  location: {城市, 国家}
  email: {邮箱}
  phone: {电话}
  website: {个人网站或 LinkedIn}
  
  sections:
    summary:
      - {3 句话：你是谁 + 最大成就 + 求职方向}
    
    experience:
      - company: {公司名}
        position: {职位}
        location: {城市}
        start_date: {YYYY-MM}
        end_date: {YYYY-MM 或 "present"}
        highlights:
          - {STAR 格式的 bullet point}
          - {每条 ≤ 25 个词，包含 1 个量化指标}
    
    education:
      - institution: {学校}
        degree: {学位}
        field: {专业}
        start_date: {YYYY}
        end_date: {YYYY}
        gpa: {GPA，如有}
    
    skills:
      - {技能分类标签，如 "Languages"}
        items:
          - {具体技能}
    
    projects:
      - name: {项目名称}
        url: {项目链接}
        highlights:
          - {成果描述}

design:
  theme: moderncv
  font: Roboto
  font_size: 10pt
  page_size: a4
  color: "#1A1A2E"
  show_timespan_in: []
  show_last_updated_date: false
```

【规则】
- 每段工作经历写 3-5 个 bullet points
- 每个 bullet 必须包含量化结果（数字/百分比/金额/时间节省）
- 不要编造数据 —— 如果不确定数字，标注 [需要数据]
- 使用过去时态描述已完成的工作
- 所有日期格式统一为 YYYY-MM

输出完整 YAML，放在代码块中。
```

### 🅱️ Phase 2a: STAR 子弹点改写

```text
你是一位 FAANG 级别的招聘经理，每天筛选 200+ 份简历。
你只给 6 秒来抓住你的注意力。

请用 STAR 方法改写以下简历 bullet points，每条限制 1 句话。

【原始 bullet】
{粘贴你的原始 bullet points}

【约束条件】
1. 以强力动作动词开头（Led, Shipped, Reduced, Generated, Built, Scaled, Optimized）
2. 包含 1 个具体量化指标（%、$、小时、用户数、收入）
3. 每条 ≤ 24 个英文词（中文 ≤ 18 字）
4. 使用过去时态
5. 严禁以下词汇：协同效应、赋能、深耕、致力于、精益求精、dynamic、synergy、spearhead、passionate、results-driven

【JD 上下文】（可选）
{粘贴目标职位 JD}

对每条原始 bullet，输出 3 个改写版本并标注：
- ⭐ = 影响力最强（优先使用）
- ✅ = 稳妥
- 💡 = 换个角度

如果一条 bullet 完全没有量化数据，标注 [需要补充数据] —— 不要编造数字。
```

### 🅲️ Phase 2b: Professional Summary 精炼

```text
用恰好 3 句话重写我的简历 Summary。每句话 1 个目的：

- 第 1 句：我的专业身份（Title + 核心领域 + 年限）
- 第 2 句：最大的可量化成就（1 个数字 + 1 个动词）
- 第 3 句：我接下来的目标方向

限制：
- 总字数 ≤ 300 字符
- 不含 " passionate ", " results-driven ", " team player "
- 不含 " 热爱 ", " 致力于 ", " 深耕 ", " 精益求精 "

我的背景：
{粘贴你的经历}

目标岗位：{JD 标题或 URL}
```

### 🅳️ Phase 3: ATS 关键词缺口分析

```text
你是一个 Greenhouse/Workday ATS 解析引擎。

请阅读以下 JD，提取 ATS 系统权分最高的关键词，
然后对比我的简历 YAML，找出缺失的关键词。

【JD】
{粘贴完整 Job Description}

【我的简历 YAML】
{粘贴你的 YAML}

分析要求：
1. 提取 JD 中权重最高的 10 个硬技能关键词（按出现频率排序）
2. 提取 JD 中重复 2 次以上的 5 个软技能关键词
3. 提取 JD 中必须出现的关键职位短语（用于 ATS title matching）
4. 标注我的简历中缺失的关键词（只列真正缺失的）
5. 对每个缺失关键词：我是否真实具备？（YES：建议如何融入 / NO：不要强行加入）

📊 最后给出 ATS 匹配度评分（0-100）：
- 70 以下 → 大概率自动拒绝
- 70-80 → 可能进入人工筛选
- 80+ → 大概率通过 ATS
- 90+ → 理想状态

⚠️ 重要：不要建议加我不具备的技能。ATS 检测到关键词堆砌（密度 > 3%）会降级。
```

## 五、设计系统：确保"高端大气有品位"

### 5.1 排版铁律

```
字体：标题 Roboto Bold 12pt / 正文 Roboto Regular 10pt
行距：1.2 倍（标题）/ 1.0 倍（正文）
间距：section 之间 12pt 空白，bullet 之间 2pt 空白
边距：上下 1.5cm，左右 1.8cm
页数：严格 1 页（10 年以下经验）或 2 页（10 年以上）
纸张：A4（亚洲/欧洲）或 Letter（北美）
```

### 5.2 配色方案（RenderCV `design.color` 参数）

```
经典商务：  #1A1A2E（藏蓝，最安全）
科技感：    #2563EB（蓝色，适合工程师）
极简黑白：  #000000（黑色，设计师/创意岗位）
温暖质感：  #B45309（茶棕，适合产品/管理）
```

**铁律**：全页只用 1 个强调色。禁止在标题行使用超过 1 种颜色。

### 5.3 信息密度黄金法则

```
✅ 大量留白 → 专业、有呼吸感
✅ 1 页原则 → 把最好的留下，其余删除
✅ 3-5 bullets per role → 再多的只保留最出彩的
❌ 满满当当 → 廉价、没人会读
❌ 10+ bullets per role → 说明你不会提炼重点
```

## 六、CLAUDE.md 配置（自动加载简历设计规则）

将以下内容添加到 `.claude/CLAUDE.md`：

```markdown
# Resume Generation Rules

## Role
You are a senior recruiter who has reviewed 50,000+ resumes and worked at
Google, Stripe, and McKinsey. You know what ATS systems parse and what
hiring managers actually read.

## Content Rules
- Every bullet must follow STAR (Situation → Task → Action → Result)
- Every bullet must include 1 quantifiable metric (%, $, users, time saved)
- Bullets ≤ 24 words (English) or ≤ 18 characters per line (Chinese)
- Summary ≤ 3 sentences
- Max 5 bullets per role (3-4 is ideal)
- Use past tense for completed work
- Never invent data — use [needs data] placeholder

## Design Rules
- 1 color accent max — no rainbow resumes
- Font: Roboto, 10pt body, 12pt section titles
- 1 page for <10 years experience, 2 pages for 10+
- Generous whitespace — if it feels crowded, delete content
- No headshot photos (unless applying in Asia/Europe where expected)
- No skill rating bars (nobody knows what "4/5 Python" means)
- No icons or progress bars — they confuse ATS

## Anti-Patterns (INSTANT REJECT)
- ❌ "Passionate about..." / "热爱..." / "致力于..."
- ❌ "Results-driven professional with a proven track record..."
- ❌ "Responsible for..." (say what you DID, not what you were assigned)
- ❌ Buzzwords: synergy, leverage, spearhead, dynamic, 赋能, 深耕
- ❌ "We" / "The team" (use "I" — take credit for your work)
- ❌ Multi-column layouts (break ATS parsing)
- ❌ Graphics, icons, skill rating bars
- ❌ References available upon request (obvious, wastes space)
```

## 八、质量自检清单

### 内容层面
- [ ] 每条 work experience bullet 包含量化数字
- [ ] Summary 恰好 3 句，不超 300 字符
- [ ] 0 个 buzzword（passionate / synergy / 热爱 / 赋能）
- [ ] 所有动词为过去时态
- [ ] 没有 "Responsible for..." / "负责..."
- [ ] 用 "I" 而非 "We"（不要隐藏个人贡献）
- [ ] 没有编造任何数据

### 设计层面
- [ ] 1 个强调色，没有彩虹配色
- [ ] 字体：Roboto（或等宽无衬线），10pt 正文 / 12pt 标题
- [ ] A4 或 Letter 页，1 页（<10年经验）
- [ ] 大量留白，不拥挤
- [ ] 没有头像、图标、进度条
- [ ] 没有多栏布局

### ATS 层面
- [ ] 文本格式（非图片/表格/文本框）
- [ ] 标题包含目标职位关键词
- [ ] Skills 区包含 JD 中的顶级硬技能
- [ ] 没有特殊字符（→ ❤ ★ 等 ATS 不识别）
- [ ] PDF 文件大小 < 500KB

---

> **最后的忠告**：简历不是你的自传，是面试的敲门砖。
> 好简历的标准不是"写得多"，而是"删得狠"。
> 每删一个废话 bullet，你的面试概率就上升一分。
> AI 帮你写 80% —— 但那 20% 的取舍判断、数据核实、最后一眼的"这个数字是真的吗"，
> 才是区分 pass 和 interview 的关键。宁缺毋滥。
