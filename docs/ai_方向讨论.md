# AI 创业方向头脑风暴 & 个人路径讨论

> 日期：2026-07-14
> 性质：全新话题讨论，与之前的 AI 测试工具项目无关
> 核心问题：AI 应用层还有哪些机会？个人如何利用 15 年工程经验 + AI 产生现金流？

---

## 一、2026 年 AI 市场全景分析

### 1.1 宏观经济判断

- AI 正从「技术竞赛」变成「交付结果的生意」
- 投资人追问的不再是「模型有多聪明」，而是「场景有多坚硬」
- **得场景者得 AI**

### 1.2 过度竞争、慎入的领域

| 方向 | 原因 |
|------|------|
| 通用 AI 写作/客服 | 无垂直壁垒，基模更新即可覆盖 |
| 通用 AI 编程助手 | Copilot/Cursor/Claude Code 三分天下 |
| 水平型 AI Copilot | 浅层封装，大厂一更新就没了 |
| GEO（生成式引擎优化）| 合规风险巨大（315 已曝光），无技术壁垒，终局被大平台吞掉。真实市场需求但**不适合独立创业** |

### 1.3 2026 年各方向详细分析

#### 高可行方向

**1. 垂直行业 AI Agent（替代高收入岗位）**

Anthropic 2026 年报告，AI Agent 调用量分布：

| 行业 | Agent 调用占比 | 行业 GDP 占比 | 鸿沟 |
|------|:--:|:--:|:--:|
| 软件工程 | 49.7% | ~5% | AI 过度集中 |
| 医疗 | 1.0% | ~18% | 几乎空白 |
| 教育 | 1.8% | ~6% | |
| 法律 | 0.9% | ~3% | |
| 金融 | <3% | ~8% | |

YC CEO 原话："那片几乎空白的区域，藏着下一代 300 个独角兽。"

具体方向：AI 法律、AI 医疗文书/诊断辅助、AI 招聘面试、AI 工业老师傅（本地知识库）

**前提：必须懂那个行业。不懂行业的人做垂直 Agent 就是灾难。**

**2. AI + 制造业（工业仿真/巡检/CAD）**

需求真实、客户有钱、数据壁垒深。但周期长，需要行业关系。

**3. 设备端 AI（On-Device AI）**

隐私需求驱动，硬件+软件一体。适合有供应链资源的团队。

#### 有风险的方向

**4. AI 短剧/视频生产** — 成本优势真实但内容行业赢家通吃

**5. 多智能体协作平台** — 大厂已占位（Coze/Dify/扣子），创业窗口窄

**6. AI 陪伴/教育** — To C 获客成本高，To B 决策链长

#### 被忽视的方向

**7. AI Agent 审计与合规** — 每个部署 AI 的企业都需要外部验证。相当于「AI 系统的 SOC 2 认证」。单次审计 $5,000-$50,000，目前几乎是空白。

## 二、个人情况分析与项目评估

### 2.1 个人基本情况

- 15 年软件工程经验
- AI 写代码能力已超越人，技术壁垒被磨平
- 目前失业状态，有危机感
- 有 3-12 个月储蓄缓冲
- 当前目标：利用 AI + 工程经验产生实际现金流

### 2.2 正在运行的项目（逐项评估）

| 项目 | 评价 | 结论 |
|------|------|------|
| **英语词根学习系统** | 英语学习 App 是红海，无获客能力 | 练手项目，别指望产生收入 |
| **Obsidian 插件（已开源）** | 正确动作（开源积累声誉）但不能直接变现 | 保留维护，每周少量时间 |
| **中小学教材知识网** | 教育行业壁垒极高，政策风险大 | 一个人做无意义，建议停掉 |
| **自媒体 4 个平台同时做** | 精力分散，变现周期 1-3 年 | 聚焦 B 站 + YouTube，放弃抖音/小红书 |
| **AI 教学视频** | 费曼学习法本身成立，定位需差异化 | 做副线，不能当主业 |

### 2.3 核心问题诊断

**根本错误模式：先做产品 → 再找用户 → 期望有人合作。**

所有项目都是「手里有锤子（AI 开发能力），到处找钉子」。但钉子不是自己想象出来的——是别人告诉你「这里有个洞需要补」。

### 2.4 最高优先：先解决现金流

**最快的变现方式不是做产品，是提供服务：**

- AI 代码审计/安全审查
- 遗留系统 AI 迁移评估
- AI Agent PoC 搭建
- 接远程 AI 开发项目

## 三、四个匹配方向分析

基于 15 年工程经验过滤后的可行方向：

| 方向 | 经验匹配度 | 市场真实度 | 变现速度 | 推荐 |
|------|:--:|:--:|:--:|:--:|
| AI 代码审计/安全审查 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 1-2 个月 | ✅ 首选 |
| AI Agent 审计合规 | ⭐⭐⭐⭐ | ⭐⭐⭐ | 12 个月+ | 市场还在萌芽 |
| 遗留系统 AI 迁移 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 2-4 个月 | ✅ 可选 |
| AI 测试工具 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 6-12 个月 | 需先做产品 |

### 关于 AI 教学视频

- 以学为目的做内容：成立（费曼学习法）
- 定位：**「15 年工程师告诉你，AI 写的代码哪些不能直接上生产」**——不是讲怎么用 AI，是讲用了 AI 之后怎么办。这个定位目前没人在做。
- 平台：B 站 + YouTube，放弃抖音/小红书/视频号
- 时机：副线做，不能当主业（没钱等不起 1-3 年变现周期）
- **最优解：内容和业务相互喂养**——审计客户案例 = 视频素材

## 四、落地执行计划

### 第一个月

**主线：搞到第一个付费客户（目标：¥2,000-5,000）**

- 找审计目标项目，做一次完整代码审计
- 写案例文章发 V2EX/掘金
- 联系 10 个前同事/前客户
- Upwork 开账号接单

**副线：第一期视频上线（不追求数据）**

- 内容 = 审计案例拆解
- B 站 + YouTube 同步

**判断标准：有没有人愿意付钱。**

### 第二到第三个月

| 场景 | 动作 |
|------|------|
| 有客户了 → | 标准化服务、做案例集、尝试提价 |
| 没客户 → | 换方向（遗留系统迁移/远程开发项目）|

### 第四到第十二个月

- 审计服务跑通 → 产品化（AI 代码审计工具）
- 视频跑起来 → 付费小课（¥99-199）
- 都没跑通 → 第四个月必须认真考虑找工作

## 五、AI 代码审计落地细节

### 5.1 审计类型

| 类型 | 内容 |
|------|------|
| 安全审计 | SQL 注入、XSS、认证绕过、敏感数据泄露、不安全依赖 |
| 性能审计 | N+1 查询、内存泄漏、不合理缓存策略、阻塞操作 |
| 架构审计 | 循环依赖、错误事务边界、不合理服务拆分 |
| 业务逻辑审计 | 边界条件遗漏、状态机缺漏、并发竞态 |

### 5.2 定价

| 档位 | 范围 | 价格 |
|------|------|------|
| 轻量审计 | 1 个核心模块（<2000行） | ¥2,000-5,000 |
| 标准审计 | 1 个微服务（<10000行） | ¥8,000-15,000 |
| 深度审计 | 整个项目/关键链路 | ¥20,000-50,000 |

### 5.3 找客户路径

1. **已有网络**（效率最高）：联系前同事/leader/客户
2. **内容获客**：V2EX/掘金发案例文章
3. **GitHub 作品集**：建 `ai-code-audit` repo，放脱敏案例
4. **自由职业平台**：Upwork/电鸭

### 5.4 第一周动作清单

| 天 | 动作 |
|----|------|
| 第 1 天 | 找一个开源项目做完整审计，写标准报告 |
| 第 2 天 | 脱敏发 V2EX/掘金 |
| 第 3 天 | 私信 10 个前同事/前客户 |
| 第 4-5 天 | 建 GitHub repo `ai-code-audit` |
| 第 6-7 天 | 跟进回复/Upwork 开账号 |

## 七、关键原则总结

1. **停止开发任何新产品** — 手头项目已过多
2. **先找客户，再做产品** — 颠覆你一直以来的「先做产品」模式
3. **一条主线，一个方向** — 不要同时跑五条线
4. **内容和业务相互喂养** — 审计案例 = 视频素材
5. **第一单可以免费，但要换 testimonial + 案例使用权**
6. **AI 省的是体力活，不是判断力** — 你的 15 年经验是产品，不是 AI 能力
