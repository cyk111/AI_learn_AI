# ATS 关键词缺口分析 Prompt

**用途**：对比简历与 JD，找出 ATS 系统会扣分的关键词缺口
**适用模型**：Claude Sonnet 4.5 / GPT-5
**关键原则**：只加真实具备的技能，杜绝关键词堆砌

---

## Prompt

```text
你是一个 Greenhouse / Workday / Lever ATS 解析引擎。
你的任务是精确分析 JD 中的关键词，然后对照简历 YAML，
找出真正的缺口 —— 不是让你堆砌关键词。

【Job Description】
{粘贴完整 JD}

【我的简历 YAML】
{粘贴你的 RenderCV YAML}

---

## Step 1: JD 反向解析

从 JD 中提取以下信息：

| 维度 | 内容 |
|------|------|
| 岗位要解决的核心问题 | {1 句话} |
| 最怕招到的人（2 个反模式） | {面试官想避免的} |
| 简历必须出现的 3 个关键信号 | {能让你脱颖而出的} |
| 硬技能 Top 10（按 JD 中出现频率） | {技术/工具/方法论} |
| 软技能 Top 5（重复 2 次以上的） | {沟通/领导力/协作} |
| 关键职位短语（用于标题匹配） | {JD 中使用的精确 Title} |
| 年限要求 | {如果有明确的 years of experience} |

---

## Step 2: 缺口分析

对照我的简历 YAML，逐项检查：

### 命中的关键词 ✅
| 关键词 | 在简历中的位置 | 匹配方式 |
|--------|---------------|---------|
| {keyword} | {在 summary / skills / experience 中} | 精确 / 语义 |

### 缺失的关键词 ⚠️
| 关键词 | JD 出现次数 | 我是否真实具备？ | 建议 |
|--------|:----------:|:-------------:|------|
| {keyword} | 3 次 | YES | 在 {具体位置} 加入 |
| {keyword} | 2 次 | NO | 不要加 — 面试会露馅 |

### 可以移除的废话 🗑️
| 当前内容 | 为什么该删 |
|---------|-----------|
| {无量化/过时/无关的内容} | {原因} |

## Step 3: ATS 匹配度评分

```
ATS Score: XX/100

├── 硬技能匹配: XX/40

├── 年限匹配: XX/10
├── 行业关键词: XX/15

评级:
  90+   🟢 大概率通过
  80-89 🟡 可能通过
  70-79 🟠 风险区间 — 建议修改后投递
  <70   🔴 建议不要投递 — ATS 大概率过滤
```

## Step 4: 定制化修改建议

给出 3-5 条具体修改建议（每条 ≤ 15 个字改动）：

1. {具体建议}
2. {具体建议}
3. {具体建议}

---

## ⚠️ 重要警告

- 只建议我真实具备的技能，不要为了匹配 JD 而加我不会的东西
- ATS 检测到关键词堆砌（某个关键词密度 > 3%）会降级扣分
- 如果我的技能和经验真的不匹配这个 JD，请诚实告知
```

---

## 使用方式

```bash
# 1. 打开 Claude / ChatGPT
# 2. 粘贴上面的 Prompt
# 3. 替换 {粘贴 JD} 和 {粘贴 YAML}
# 4. 得到缺口分析报告
# 5. 根据建议微调 YAML（只改真实具备的）
# 6. rendercv render my_cv.yaml
```
